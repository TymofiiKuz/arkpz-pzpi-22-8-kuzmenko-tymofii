1 Лабораторна робота №1: РОЗРОБКА VISION & SCOPE


1.1 Аналіз проблеми, яку вирішує програмна система
Актуальність проблеми безпеки на дорогах зростає у зв'язку зі збільшенням кількості транспортних засобів та дорожніх пригод. Щороку велика кількість аварій трапляється через людські помилки, несправність автомобілів або недосконалість інфраструктури. Статистичні дані свідчать про те, що значна частина аварій могла б бути попереджена при використанні сучасних технологій моніторингу стану автомобіля та дорожньої ситуації.  


Програмна система "Безпека автомобіля" допомагає знизити ризики аварій за допомогою автоматизованого відстеження технічного стану транспортного засобу, попередження про потенційні загрози на дорозі, а також забезпечення своєчасного втручання в критичних ситуаціях (активування гальмівної системи або регулювання швидкості). Таким чином, ця система спрямована на підвищення безпеки водія, пасажирів та інших учасників дорожнього руху.


1.2 Аналіз аналогів
На ринку вже існують подібні рішення для підвищення безпеки автомобілів, такі як:
- Системи допомоги водієві (ADAS), що включають функції автоматичного екстреного гальмування, попередження про вихід з смуги руху та виявлення перешкод.
- Tesla Autopilot – система часткового автопілоту, яка забезпечує автоматизоване керування транспортним засобом.
- Системи моніторингу стану автомобіля: інтегровані рішення від виробників, як-от OnStar (General Motors) або BMW Connected Drive, що пропонують віддалений моніторинг автомобіля та діагностику.


Програмна система "Безпека автомобіля" має конкурентну перевагу за рахунок інтеграції IoT-технологій, персоналізації повідомлень, а також можливості відстежувати стан автомобіля в реальному часі з мобільного додатку.


Способи монетизації проєкту:
1. Передплатна модель: місячна або річна підписка на систему безпеки з можливістю отримання додаткових функцій (аналітика, історія поїздок, персональні рекомендації).
2. Інтеграція з автовиробниками: продаж ліцензій або готового рішення для інтеграції в нові автомобілі.
3. Продаж IoT-пристроїв: датчики та інші пристрої, що можуть бути встановлені на автомобіль для розширення функцій системи.


2.1 Опис призначення системи
Програмна система "Безпека автомобіля" призначена для моніторингу технічного стану автомобіля, попередження про небезпечні ситуації на дорозі та допомоги водіям в уникненні аварійних ситуацій. Вона дозволяє аналізувати роботу транспортного засобу, надавати рекомендації водієві та автоматично втручатися у критичних ситуаціях. 


2.2 Опис бізнес-логіки системи
Основні бізнес-процеси системи включають:
- Моніторинг у реальному часі: система збирає дані з датчиків автомобіля (швидкість, тиск у шинах, рівень палива, стан двигуна) та аналізує їх.
- Автоматичні попередження: водій отримує повідомлення про несправності, перевищення швидкості або небезпечні умови на дорозі (обледеніння, інші транспортні засоби).
- Автоматичне втручання: у разі критичних ситуацій система може самостійно активувати гальма або зменшити швидкість для уникнення аварії.
- Звіти та аналіз: надаються докладні звіти про поїздки та рекомендації щодо покращення безпеки водіння.


3.1 Опис функціональності серверної частини та IoT клієнта
- Серверна частина: обробляє та зберігає дані, що надходять з IoT-пристроїв і автомобіля. Сервер відповідає за обробку великих обсягів даних у реальному часі та надання аналітичних звітів.
- IoT-клієнт: включає датчики та інші пристрої, які збирають дані про стан автомобіля (швидкість, тиск у шинах, паливо тощо) і передають їх на сервер. IoT-клієнт також забезпечує взаємодію з мобільними пристроями водія для оперативного надання інформації.


3.2 Опис функціональності веб клієнта та мобільного клієнта
- Веб клієнт: забезпечує інтерфейс для доступу до звітів та аналітики через веб-браузер. Водії можуть переглядати історію поїздок, аналізувати поведінку на дорозі та отримувати рекомендації.
- Мобільний клієнт: дає можливість отримувати реальні повідомлення про стан автомобіля, аварійні сповіщення та рекомендації щодо водіння через смартфон або смарт-годинник. Мобільний додаток також дозволяє віддалено перевіряти стан автомобіля.


4. Опис програмних засобів та інструментів, які будуть використані у проекті
- Мови програмування: Python (для серверної частини), JavaScript (для веб-інтерфейсу), Swift (для iOS) та Kotlin (для Android).
- IoT технології: використання протоколу MQTT для передачі даних з датчиків, інтеграція з апаратним забезпеченням через Raspberry Pi або Arduino.
- Бази даних: MongoDB або PostgreSQL для зберігання даних про поїздки та технічний стан автомобіля.
- Хмарні технології: AWS або Microsoft Azure для масштабованого хостингу серверної частини системи.
- Фреймворки для фронтенду: React або Angular для веб-інтерфейсу.
- Мобільні фреймворки: використання нативних рішень для Android та iOS або кросплатформеного рішення Flutter для мобільних додатків.


 Строки виконання проєкту
- Аналіз вимог та проєктування: 1 місяць
- Розробка серверної частини та IoT клієнта: 2 місяці
- Розробка веб клієнта та мобільного додатку: 3 місяці
- Тестування та інтеграція: 1 місяць
- Запуск і підтримка: 1 місяць


 Зацікавлені сторони
1. Автовиробники, зацікавлені у підвищенні безпеки своїх автомобілів.
2. Водії, які бажають підвищити рівень безпеки під час водіння.
3. Страхові компанії, які можуть отримати інструменти для зниження ризиків та аварій.
4. Транспортні компанії, що прагнуть оптимізувати безпеку своїх автопарків.
